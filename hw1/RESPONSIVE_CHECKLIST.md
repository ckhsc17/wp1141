```css
/* 手機 (預設) */
default: 0px

/* 小平板 */
sm: 640px

/* 平板 */
md: 768px  

/* 小筆電 */
lg: 1024px

/* 桌機 */
xl: 1280px

/* 大螢幕 */
2xl: 1536px
```

## 響應式設計原則

### 1. Mobile-First 設計
- 所有樣式先針對手機設計
- 使用斷點向上擴展到更大螢幕
- 確保核心功能在所有裝置上都能正常運作

### 2. 彈性佈局
- 使用 Flexbox 和 Grid 進行響應式佈局
- 避免固定寬度，優先使用相對單位
- 適當的間距在不同螢幕尺寸下保持視覺平衡

### 3. 內容優先級
- 重要內容在小螢幕上優先顯示
- 非必要元素在小螢幕上隱藏或簡化
- 保持良好的可讀性和可用性

## 測試指南

### 手動測試斷點
建議在以下寬度進行測試：
- 320px (小手機)
- 375px (iPhone SE)  
- 768px (iPad 直向)
- 1024px (iPad 橫向)
- 1280px (小筆電)
- 1920px (桌機)

### 瀏覽器開發者工具
1. 開啟 Chrome DevTools
2. 點擊 "Toggle device toolbar" (Ctrl+Shift+M)
3. 測試不同預設裝置
4. 使用 "Responsive" 模式手動調整寬度
